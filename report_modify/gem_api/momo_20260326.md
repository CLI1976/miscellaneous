# 開發紀錄 2026-03-26

## 目標
將 `gem.py`（Gemini API 放射科報告工具）的輸出改為可 select/copy 的 GUI 視窗，並整合 AutoHotkey 快捷鍵觸發。

---

## 過程摘要

### 1. GUI 選型
- **tkinter** → PortablePython 無此模組，無法 pip 安裝
- **瀏覽器開 HTML** → 可用但啟動慢
- **PyQt6 + QTextBrowser** → 安裝成功，但 HTML/CSS 渲染能力有限，排版難以控制
- **PyQt6 + QWebEngineView** → 最終選擇，完整 Chromium 引擎，渲染品質等同瀏覽器

### 2. AHK 整合（Gem_report.ahk）
- 修正結構 bug：`ToolTip` / `RunPythonBrowser` 原本在 F4 block 外，啟動時就執行
- 快捷鍵改為 F2，限定 Notepad++ 為 active 視窗才觸發（`#HotIf`）
- 報告文字改用 temp file 傳遞，避免 command line 引號跳脫問題
- Tooltip 改為非同步：Python 視窗開啟後寫 `gem_ready.txt`，AHK Timer 偵測到即關閉

### 3. 排版問題的根本原因
花了大量時間處理 Markdown 渲染問題：
- `nl2br` extension 會破壞 nested list 解析
- Python `markdown` 需要 list 前有空行才能正確解析
- 加了三個 post-processing 函式（`fix_inline_lists` / `fix_list_nesting` / `ensure_blank_before_list`）
- 根本原因：**model 輸出格式不穩定**，每次呼叫可能用不同格式輸出 bullet list

### 4. 架構改為 JSON 輸出（最終解法）
- 不再要求 model 輸出 Markdown，改為要求輸出**嚴格 JSON**
- Python 解析 JSON 後自行生成 HTML，格式 100% 可控
- 移除全部 post-processing 函式與 `markdown` 套件依賴
- JSON schema 包含：`intro` / `report`（findings + impression）/ `dd`（鑑別診斷）/ `note`

### 5. 其他細節
- 字型：Microsoft JhengHei 優先，大小可調（`FONT_SIZE`）
- 背景：淺灰色（`#f5f5f5`）
- 連結點擊：自訂 `QWebEnginePage` 開啟系統瀏覽器
- API Key 移出程式碼，改存於 `api.ini`
